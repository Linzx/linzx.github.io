
### 1. 资源泄漏
当申请的资源没有及时正确地释放，势必导致资源泄漏，长时间运行则可能出现资源耗尽而异常Crash.
下面列举一些成对的资源申请/释放。

- open -> close
- fopen -> fclose
- malloc -> free
- new -> delete

### 2. 异常处理

- 可能出错的语句，要有异常处理逻辑：
  - 网络请求
  - 数据库交互
  - 文件操作
- 异常条件的判定
- 边界条件的考虑 
- 数组越界
- 缓冲区溢出

### 3. 多线程问题

多线程同时读写同一个资源， 比如变量，文件，缓存区。

- 遇到资源读写，则检查同步锁
- 注意加锁范围，不可过大，也不可过小
